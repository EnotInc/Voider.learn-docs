# 1. Обзор продукта
## 1.1 Название проекта
Рабочие название - "Voider.learn". В дальнейшем может потерпеть изменения.
## 1.2. Назначение проекта
Образовательная платформа для изучения основ программирования, алгоритмов, структур данных и для подготовки к первому собеседованию. Платформа вдохновлена Duolingo и во многом следует по ее стопам: короткие интерактивные уроки, система достижений и прогресса по курсу.
## 1.3. Целевая аудитория
Начинающие разработчики, студенты IT-специальностей, а так же все желающие познакомится с программированием.
# 2 Бизнес-цели
## 2.1. Привлечение и удержание пользователей
Увлекательный продукт позволяет удерживать пользователей и мотивирует их возвращаться в приложение каждый день
## 2.2. Формирование образовательной привычки
Пользователь будет постепенно чему-то учиться или закреплять уже полученные навыки, что позволит ему вырасти качественно и получить полезные навыки
## 2.3. Возможности монетизации
Добавление рекламы, за просмотр которой пользователь будет получать различные бонусы внутри приложения. Добавление подписки, отключающей рекламу и открывающей дополнительные функции.
# 3. Пользователи и сценарии
## 3.1. Пользовательские роли
- **Анонимный пользователь**: Не имеет доступа к контенту внутри приложения
- **Зарегистрированный пользователь**: Может проходить уроки, отслеживать прогресс обучения, получать ачивки, редактировать свой профиль
- **Администратор**: Имеет доступ к статистике пользователей, может блокировать пользователей с подозрительной активностью
## 3.2. Сценарий использования
### 3.2.1. Регистрация
1. Пользователь заходит в приложение
2. Вводит логин, email, пароль(х2)
3. Нажимает "Зарегистрироваться"
4. Попадает в "приветственный урок"
5. После прохождения урока, пользователю доступен весь функционал приложения
### 3.2.2. Прохождение урока
1. Пользователь выбирает доступный урок
2. Изучает вопрос, код/схему
3. Выбирает один из данных ответов, или вводит свой
4. Нажимает "Ответить"
5. При неправильном ответе, пользователю дается возможность ввести иной ответ
6. За один урок пользователь отвечает на 5-10 вопросов
7. После урока пользователю может быть предложено прочитать "Дополнительные материалы"
### 3.2.3. Отслеживание прогресса обучения
1. В главном меню пользователю доступна "дорожная карта" обучения, и его текущее местоположение
2. Пользователь переходит в профиль
3. Нажимает на "Ачивки" -> получает список достижений

# 4. Функциональные требования
## 4.1. Модуль регистрации/авторизации
- При первом запуске пользователю предлагается зарегистрироваться, или авторизироваться в приложении(при наличии аккаунта), используя email \ login и парль
- При последующих запусках приложения данное окно не появляется до тех пор, пока пользователь самостоятельно не выйдет из аккаунта через "профиль"
# 4.2. Домашняя страница(дорожная карта)
- Уроки, Последовательно выстроенные в "дорожную карту"
- Пользователь может выбрать любой из уже пройденных уроков, или следующий урок.
- Пользователь может перейти к след. разделу, тогда ему нужно будет пройти тест на знание пропущенного раздела
- Каждый урок содержит:
	- Задачи (5-10)
		- Описание задачи
		- Доступные варианты ответа / поле для ввода ответа
		- Автоматическую проверку ответа
		- Подсказки(опционально)
	- Теория (1)
		- Может быть только 1 раз перед задачами(опционально)
		- Содержит краткое описание деталей урока
		- Ссылается на внешние источники(опционально)
		- Визуальное представление описываемых деталей(Опционально)
## 4.3. Профиль
- Содержит Аватар пользователя
- Список достижений
- Поле с настройками
	- "Логин"
	- "Почта"
	- "Пароль"
	- "Выход из аккаунта"

## 5 Нефункциональные требования
## 5.1. Производительность
- Время загрузки не должно превышать 10 секунд
- Время проверки ответа не должно превышать 1 секунду
## 5.2. Удобство использования
- Интуитивно понятный интерфейс: Простой, чистый, без лишней сложности
- Единообразие дизайна внутри модулей приложения
## 5.3. Надежность
- Система должна корректно обрабатывать ошибки
## 5.4. Безопасность
- По возможности реализовать защиту от самых распространенных уязвимостей (owasp top 10)
- Пароли должны храниться в хешированном виде
# 6. Архитектура и стек
## 6.1. Общая архитектура
Клиент-серверная архитектура c REST API
## 6.2. Стек технологий
- Клиент (Frontend)
	- Futter(dart) - Основной фреймворк для разработки приложения под Android
- Серверная часть (Backend)
	- Node.js - Основная платформа
	- Express.js - Фреймворк для написания API
	- PostgreSQL - База данных пользователей, их прогресса, курсов и всего остального

# 8. API 

Список endpoint-ов:
1. **POST** /api/v1/auth/register - регистрация нового пользователя
2. **POST** api/v1/auth/login - логин существующего пользователя
3. **GET** api/v1/home/roadmap - дорожная карта уроков пользователя
4. **GET** api/v1/home/courses - список курсов
5. **POST** api/v1/home/courses - запись пользователя на курс
6. **DELETE** api/v1/home/courses - удаление курса у пользователя
7. **PUT** api/v1/lessons/begin - получение данных о уроке
8. **PUT** api/v1/lessons/done - завершение урока
9. **GET** api/v1/profile - данные о профиле пользователя
10. **PUT** api/v1/profile - изменение данных профиля
11. **GET** api/v1/profile/achivements - список достижений пользователя
12. **POST** api/v1/profile/achivements - добавление достижения
